# 層③：セルフレビュープロンプト（品質チェック）

## 使用タイミング
層②で問題を生成した直後に、同じセッション内で以下のプロンプトを使い品質チェックを行う。

## レビュープロンプト

```
以下のTypeScript問題を品質チェックしてください。

{生成した問題のJSON}

以下の観点で各問題を評価し、問題があれば修正案を出してください。

1. 【正確性】TypeScript 5.x の仕様として正しいか
2. 【難易度の妥当性】以下の基準に合っているか
   - difficulty 1：知っていれば解ける
   - difficulty 2：なぜそうなるかの理解が必要
   - difficulty 3：設計・トレードオフの判断が必要
3. 【選択肢の品質】4択全てが「ありえそう」に見えるか・明らかな間違いがないか
4. 【解説の完全性】正解の理由と他の選択肢が間違いの理由が両方説明されているか
5. 【コードの正確性】コードブロックが意図通りに動作するか（コンパイルエラーの有無）

出力形式（JSONのみ）：
{
  "passed": ["問題IDのリスト"],
  "failed": [
    {
      "id": "問題ID",
      "issues": ["問題点1", "問題点2"],
      "fixed": { 修正後の問題JSONオブジェクト }
    }
  ]
}
```

## レビュー結果の処理
- `passed`の問題：そのまま`src/data/questions.json`に追記する
- `failed`の問題：`fixed`の内容で置き換えてから追記する
- `failed`が3問以上ある場合：バッチ全体を再生成することを検討する
