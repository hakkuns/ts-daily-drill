# TypeScriptコードレビュースキル

## レビュー観点（優先順）

### 1. 型安全性
- `any`の使用箇所（`unknown`への置き換えを提案）
- 型アサーション（`as`）の乱用
- `null` / `undefined` の未考慮
- 戻り値の型アノテーション漏れ

### 2. TypeScript固有のアンチパターン
- Java的なクラス設計（関数・型合成で解決できる場面）
- `==` vs `===`（型強制の問題）
- 型ガードが必要な場面での未使用
- Discriminated Unionで解決できる条件分岐

### 3. 非同期処理
- `await`忘れ
- エラーハンドリング不足（try-catchなし）
- `Promise.all`で並列化できる直列処理
- `res.ok`チェックなしのfetch

### 4. Reactコード固有
- 不必要な再レンダリング（useCallback/useMemo の過不足）
- useEffectの依存配列の誤り
- stateの直接変更（イミュータブル更新の原則）
- Hooksのルール違反

### 5. 設計・可読性
- SRP違反（1クラス・1関数が複数の責務を持つ）
- 命名の曖昧さ
- マジックナンバー・マジックストリング

## Java経験者への補足コメント
- Java的な書き方をTypeScriptらしい書き方に変更する場合は、その理由を説明する
- 「Javaではこうだが、TypeScript/JavaScriptではこう考える」という視点で説明する
